HTML Layout & Semantic Structure

Learn how to structure your page using modern HTML5 layout tags instead of only <div>.

Level: Beginner Estimated Time: 15 min Track: Web Development

Block vs Inline Elements

In HTML, elements are mainly of two display types:

  • Block elements – start on a new line and take full width (e.g. <div>, <p>, <h1>, <section>).
  • Inline elements – stay inside the text flow (e.g. <span>, <a>, <strong>). 
<p>This is a block element</p>
<a href="#">This is inline</a> text continues...

HTML5 Semantic Layout Tags

Instead of using only <div>, we use semantic tags to describe sections:

  • <header> – top area, logo, navigation
  • <nav> – navigation links
  • <main> – main content of the page
  • <section> – logical sections of content
  • <article> – independent content (blog post, article)
  • <aside> – side content (sidebar, related items)
  • <footer> – bottom area, copyright, links

Basic Page Layout Example

Here is a simple layout structure using these tags:

<body>
  <header>
    <h1>My Website</h1>
    <nav>
      <a href="#">Home</a>
      <a href="#">About</a>
    </nav>
  </header>

  <main>
    <section>
      <h2>Latest Article</h2>
      <article>
        <h3>Article Title</h3>
        <p>Article content...</p>
      </article>
    </section>

    <aside>
      <h3>Sidebar</h3>
      <p>Links or extra info</p>
    </aside>
  </main>

  <footer>
    <p>© 2025 My Website</p>
  </footer>
</body>

How This Jayanth Docs Page Uses Layout

Your current project already uses a layout similar to this:

  • Top bar is like a <header> with brand and <nav>.
  • Sidebar is like an <aside> with tutorial links.
  • The big content area is an <article> inside <main>.
  • Bottom area is a <footer> with copyright text.

Later, when you move to React or other frameworks, this mental model of “header / main / sidebar / footer” will be very useful.

Semantic Layout – Why It Matters

  • Helps search engines understand your page.
  • Improves accessibility for screen readers.
  • Makes your HTML code easier to read and maintain.
