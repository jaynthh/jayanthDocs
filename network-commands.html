Network Commands

Learn the most important commands used by IT support engineers to troubleshoot network issues.

Level: Beginner Use Case: L1 Support

Why These Commands Matter

These commands help you quickly identify network issues like connectivity problems, DNS failures, and IP configuration errors.

1. ipconfig

ipconfig

Displays IP address, subnet mask, and default gateway.

ipconfig /all

Shows detailed network configuration.

2. ping

ping 8.8.8.8

Checks network connectivity.

ping google.com

Checks DNS + connectivity.

3. nslookup

nslookup google.com

Checks DNS resolution.

4. tracert

tracert google.com

Shows the path packets take to reach a server.

5. netsh (Network Reset)

netsh winsock reset
netsh int ip reset

Resets network configuration (used in major issues).

6. Flush DNS

ipconfig /flushdns

Clears DNS cache.

Real Troubleshooting Flow (L1)

Step 1: ipconfig → Check IP
Step 2: ping 8.8.8.8 → Check network
Step 3: ping google.com → Check DNS
Step 4: nslookup → Verify DNS
Step 5: netsh reset → Fix issues

💡 Real L1 Tip

Don’t randomly try commands. Follow a logical sequence. Understanding the output is more important than running the command.
