ITSM Core

Incident Management

Learn how IT support teams quickly restore services when issues occur, using structured incident handling workflows.

Level: Beginner Estimated Time: 10 min Track: IT Support / ITSM

What is an Incident?

An incident is any unplanned interruption or reduction in the quality of an IT service.

Examples:

  • No internet connection
  • System not booting
  • Email not working
  • Application crash

Goal of Incident Management

The primary goal is to restore normal service operation as quickly as possible with minimal impact on business operations.

Incident Lifecycle

  1. Identify: User reports issue
  2. Log: Ticket created in system
  3. Categorize: Type of issue
  4. Prioritize: Based on impact & urgency
  5. Assign: To support team
  6. Resolve: Fix the issue
  7. Close: Confirm with user

Priority Levels

P1 - Critical (System down)
P2 - High (Major functionality issue)
P3 - Medium (Partial issue)
P4 - Low (Minor issue)

Real-World Example (L1 Support)

User reports: "No Internet"

Step 1: Check network cable
Step 2: Restart system
Step 3: Run ipconfig /renew
Step 4: Escalate if unresolved
