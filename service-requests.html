ITSM Core

Service Requests

Learn how IT teams handle routine user requests like password resets, access requests, and software installations in a structured way.

Level: Beginner Estimated Time: 8 min Track: IT Support / ITSM

What is a Service Request?

A service request is a formal request from a user for something they need — not a failure or issue.

Examples:

  • Password reset
  • New software installation
  • Access to applications
  • New laptop request

Incident vs Service Request

Incident = Something is broken
Service Request = Something is needed

Service Request Workflow

  1. Raise Request: User submits ticket
  2. Approval: Manager approval (if required)
  3. Fulfillment: IT team processes request
  4. Closure: Confirm with user

Real-World Example (L1 Support)

User requests: "Reset my password"

Step 1: Verify user identity
Step 2: Reset password in system
Step 3: Share temporary password
Step 4: Ask user to change password after login

Why Service Requests Matter

  • Standardized request handling
  • Improves efficiency
  • Better tracking and reporting
  • Enhances user satisfaction
