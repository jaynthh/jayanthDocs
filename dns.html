DNS (Domain Name System)

Understand how domain names are converted into IP addresses and how DNS issues are handled in real-world IT support.

Level: Beginner Track: Networking

What is DNS?

DNS (Domain Name System) is like the internet’s phonebook.

It converts human-friendly domain names (like google.com) into IP addresses (like 142.250.x.x) that computers understand.

Example

google.com → 142.250.183.14

How DNS Works (Simple Flow)

  1. User enters website (google.com)
  2. System asks DNS server
  3. DNS returns IP address
  4. Browser connects to server

Useful DNS Commands

Check DNS resolution:

nslookup google.com

Flush DNS cache:

ipconfig /flushdns

DNS Troubleshooting (Important)

ping 8.8.8.8  → Works (Internet OK)
ping google.com → Fails (DNS issue)

If IP ping works but domain fails → DNS problem.

Common DNS Issues

  • Website not opening
  • DNS server not responding
  • Slow browsing

Real-World Use (L1 Support)

When a user says "Internet is not working", DNS is often the issue. 

Step 1: ping IP
Step 2: ping domain
Step 3: flush DNS
Step 4: change DNS (8.8.8.8)

💡 Real L1 Tip

Always test both IP and domain. That quickly tells whether the issue is network or DNS.
